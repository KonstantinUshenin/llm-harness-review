# llm-harness-review
Short notes about LLM harness architecture and design

# Русский (ru_RU)

## Claude code (leaked code)

### Общие заметки

* Проект написан на TypeScript
* Используется фреймвор React, дли визуализации в "консольном варианте" библиотека Ink
* Шаблоны промптов раскинуты по файлам. Они не собраны в одном месте, паттерн менеджера (PromptManager) не используется.
* Базы данных в классическом понимании нет, все результаты складируются в файловую систему
* В проекте используется голый протокол OpenTelemetry, без надстроек

Защита от касяков:
* Скилл `batch` запускается только на файловой системе с репозиторием.
* Некоторые операции тербующие чтения запускаются параллельно, а изменения вносятся в основном потоке.
* В тулах для bash и PowerShell есть явная защита от вызова сна (sleep)

Прочее:
* В отличие от предложений LangFuse, PromptManager не используется и промпты раскиданы по файлам и локализованы в местах где они нужны. Промпты фиксированы и хранятся в GIT.
* Структурированного вывода почти нет

Промпт инжинеринг:
* Маленькие и простые регулярки пишутся прямов промптах
* Импользуются 6-shot примеры, с сильно неперекрывающимися форматами сообщений
* Аналогично argparse позволяет создавать команды с нечётким порядком аргументов. И соответственно оборачивать консольные тулы в скиллы
* Классическая петля Split-Map-Reduce через промпт
* Используются блоки (**Success criteria**) для задания успешности выполнения задачи
* Для того чтобы показать нежелательные действия not пишется большими буквами как `NOT`.

Система тегов:
* Внутри есть парсинг xml тегов для сообщений.
* В промптах поддерживается система xml тегов. Они явно прописаны в разделе `./constants/xml.js`
* Они разбираются на блоки при анализе промпта.
* Через них реализована система режимов работы, сообщений между агентами роя.


### Skills

Термины:

* Bundled skills -- навыки которые поставляются сразу с Claude code
* Disk based skills -- навыки для которых нужны файлы на диске

Skills:

* Скиллы реализованы в виде паттерна Registry. Один файл -- один скилл.
* Внутри скиллов нет обращения к AppState, но есть работа с файловой системой
* В объекте скиллов есть поле `files?: Record<string, string>` (объект `./src/skills/BundledSkillDefinition`), оно определяет файлы сопутствующие скиллу. Они будут загружены в специальную папку при инициализации.

Скилл `batch`:

* Запускает параллелизуемые работы.
* Сначала входит в режим планирования. Разделяет задачи по субагентам и просит сделать все задачи независимыми, так чтобы рабочие деревья GIT не перекрывались
* Так же вызывает количество субагентов и строит таблицу для отображения прогресса по всем скиллам.

Скилл `simplify`:

* Осуществляет проверку кода и и справляет ошибки
* Запускает трёх агентов параллельно, потом аггрегирует их вывод и исправляет найденные ошибки в основном потоке
* Что делают агенты. 1 -- Code Reuse Review, поиск дублирования кода (). 2 -- Code Quality Review, качество кода и детекция переусложнений. 3 -- Efficiency Review, возможные утечки памяти и лишние вычисления.
* Задачи для параллельного и последовательного исполнения де факто прописаны в виде теков Markdown

Скилл `loop`:

* Даёт команду `/loop`
* По факту оборачивает системную команду в разбор аргументов, анализ что нужно вызывать, постановку задачи в prompt.
* Позволяет ставить пропт, скилл или консольную команду на исполнение в cron
* Нет инструмента чтобы потом это всё из cron почистить

Скилл `remember`:
* Проверяет и исправляет память
* Работает в основном процессе, не разделяет на субагентов
* Всё что есть в памяти делит на 4 класса: Promotions -- то что переносить, Cleanup -- то что нужно удалить, Ambiguous -- то что нужно спросить у пользователя, No action needed -- ничего не делать.
* Старается не класть в память настройки IDE и окружения. Если возникает конфликт старается приоретизировать более поздние знания.
* Де факто, это система персонализации

Скилл `claudeApi`:

* Помогает пользователю с написанием кода для Antrophic API
* Детектирует язык и на основании этого передаёт в контекстное окно перенаправления на документацию для стандартных сценариев использования
* Тул содержит команды 'Read', 'Grep', 'Glob', 'WebFetch' для получения поддержки по вопросам
* В описании скилла явно написано что не надо вызывать скилл на OpenAI API

Скилл `claude-in-chrome`:
* Оборачивает MCP сервер claude-in-chrome, чтобы явно выдавать пользователю сообщение о начале использования этих тулов


#### Только для пользователя `ant`

Скилл `ant`:

* Пытается понять почему низкая проиводительность или фризы
* Анализирует список процессов системы
* Если найдёт тул для Slack, то закинет сообщение в Slack на сервер антропик в `#claude-code-feedback`. Если не найдёт -- то запишет ответ в выводе.

Скилл `debug`:

* Читает свои логи, пытается предложить решение для починки
* Вызывает субагента, который определяет фичи и состояние клод кода. Анализирует запрос пользователя и вызывает grep по логам для ускорения работы.

Скилл `lorem-ipsum`:

* Генерирует огромное количество токенов в контекстное окно
* Этот скилл не содержит промпт и реализован как функция

### Инструменты

Скилл `sleep`:

* Выполняется когда пользователь просит ничего не делать
* Ищет полезную работу которую можно сделать в отсутсвии пользователя и запускает её выполнение
* Приоритет над вызовом sleep из bash, так как они похожи по сементике
* `prompt cache expires after 5 minutes of inactivity` -- ?
* В тулах для bash и PowerShell есть явная защита от вызова сна
* Обрабатывает `tick`
